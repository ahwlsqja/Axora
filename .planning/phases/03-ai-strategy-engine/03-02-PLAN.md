<task type="auto">
  <name>Task 1: Create POST /api/strategy route handler</name>
  <files>src/app/api/strategy/route.ts</files>
  <action>
Create `src/app/api/strategy/route.ts`:

1. Set `export const runtime = 'nodejs'` at top -- Injective SDK needs Node.js for gRPC, NOT Edge runtime.

2. Import:
   - `generateText`, `Output`, `NoObjectGeneratedError` from 'ai'
   - `getModel` from '@/lib/ai/provider'
   - `strategyProposalSchema` from '@/lib/ai/schemas'
   - `buildSystemPrompt`, `getPromptForIntent` from '@/lib/ai/prompts'
   - `fetchMarketSnapshot` from '@/services/injective/spot'
   - `validateProposal` from '@/lib/strategy/validator'
   - `NextResponse` from 'next/server'

3. `POST` handler:
   - Parse request body: `{ presetId, freeText, source, marketId }`
   - Validate inputs: source must be 'preset' or 'freetext', marketId must be non-empty string, freeText truncated to 500 chars
   - Fetch market snapshot via `fetchMarketSnapshot(marketId)`
   - Build user prompt via `getPromptForIntent(presetId, freeText, source)`
   - Call `generateText` with:
     - `model: getModel(source)`
     - `system: buildSystemPrompt(market)`
     - `prompt: userPrompt`
     - `output: Output.object({ schema: strategyProposalSchema })`
   - If `result.output` is null, return 500 with error message
   - Post-validate: `validateProposal(result.output, market)`
   - Return JSON: `{ proposal: result.output, validation, market: { midPrice, bestBid, bestAsk } }`
   - Catch `NoObjectGeneratedError` -> return 422 with `{ error: 'AI could not generate a valid strategy', details: error.text }`
   - Catch other errors -> return 500 with generic error message (do NOT leak error details to client)
  </action>
  <verify>
    Run `npx tsc --noEmit` to confirm the route compiles. Verify the file exports `POST` and `runtime`.
  </verify>
  <done>
    POST /api/strategy route handler exists, compiles, uses Node.js runtime, calls LLM with schema validation, post-validates against market data, and returns structured StrategyGenerationResponse. Error handling covers NoObjectGeneratedError (422) and generic errors (500).
  </done>
</task>

<task type="auto">
  <name>Task 2: Create strategy store and React Query hooks</name>
  <files>
    src/stores/strategyStore.ts
    src/hooks/useStrategy.ts
    src/hooks/useMarketData.ts
  </files>
  <action>
1. Create `src/stores/strategyStore.ts`:
   - Import `create` from 'zustand'
   - Import `StrategyProposal` from '@/lib/ai/schemas'
   - Import `ValidationResult` from '@/lib/strategy/types'
   - Import `recalculateOrders` from '@/lib/strategy/calculator'
   - State: proposal (StrategyProposal | null), validation (ValidationResult | null), isGenerating (boolean), error (string | null)
   - Actions:
     - `setProposal(proposal, validation)`: sets both, clears isGenerating and error
     - `setGenerating()`: sets isGenerating true, clears error
     - `setError(msg)`: sets error, clears isGenerating
     - `adjustSplitCount(count: number)`: recalculates orders using `recalculateOrders` with current proposal's strategyType, priceRange, new splitCount, and totalCapitalRequired. Updates proposal.orders, proposal.splitCount.
     - `adjustPriceRange(min: number, max: number)`: recalculates orders with new range. Updates proposal.priceRange and proposal.orders.
     - `adjustTotalAmount(amount: number)`: recalculates orders with new total. Updates proposal.totalCapitalRequired and proposal.orders.
     - `reset()`: clears all state to initial values
   - Key: ALL adjustment actions use `recalculateOrders` for deterministic client-side recalculation. NO AI re-call.

2. Create `src/hooks/useStrategy.ts`:
   - Import `useMutation` from '@tanstack/react-query'
   - Import `useStrategyStore` from '@/stores/strategyStore'
   - Import `StrategyGenerationRequest`, `StrategyGenerationResponse` from '@/lib/strategy/types'
   - `useStrategy()` hook:
     - Get `setProposal`, `setGenerating`, `setError`, `reset` from strategyStore
     - Create mutation that POSTs to '/api/strategy' with StrategyGenerationRequest body
     - `onMutate`: call `setGenerating()`
     - `onSuccess`: call `setProposal(data.proposal, data.validation)`
     - `onError`: call `setError(error.message)`
     - Return: `{ generate: mutation.mutate, isGenerating: mutation.isPending, error: mutation.error, reset }`

3. Create `src/hooks/useMarketData.ts`:
   - Import `useQuery` from '@tanstack/react-query'
   - `useMarketData(marketId: string | null)` hook:
     - Fetches GET /api/market?id={marketId} -- BUT since we don't have a separate market API, instead make this a simple wrapper that fetches from the strategy API's market data or uses a dedicated lightweight endpoint.
     - Actually, simpler approach: create a thin GET handler or just use the data returned from the strategy generation response. For MVP, this hook will call a small GET /api/market endpoint.
     - Create `src/app/api/market/route.ts` as a bonus file: GET handler that takes `?id=marketId` query param, calls `fetchMarketSnapshot(marketId)`, returns the MarketSnapshot. Set `runtime = 'nodejs'`.
     - The hook uses `useQuery({ queryKey: ['market', marketId], queryFn: ..., enabled: !!marketId, staleTime: 30_000 })` with 30s cache.
     - Returns `{ data: MarketSnapshot | undefined, isLoading, error }`
   - `useSupportedMarkets()` hook:
     - Fetches GET /api/market/supported -- OR simpler: call `getSupportedMarkets()` via a GET endpoint.
     - Create `src/app/api/market/supported/route.ts`: GET handler returning supported markets map.
     - Hook uses `useQuery({ queryKey: ['supported-markets'], queryFn: ..., staleTime: 5 * 60_000 })` with 5min cache.

   NOTE: The two small GET API routes for market data (src/app/api/market/route.ts and src/app/api/market/supported/route.ts) should also be created in this task since they are lightweight and the hooks depend on them. Both set `runtime = 'nodejs'`.
  </action>
