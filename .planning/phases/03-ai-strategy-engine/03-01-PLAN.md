  <action>
1. Install dependencies:
   ```
   npm install ai @ai-sdk/openai zod
   ```
   Use Zod v3 (NOT v4) -- AI SDK v6 depends on zod-to-json-schema which targets v3.

2. Create `src/lib/strategy/types.ts`:
   - `MarketSnapshot` interface: marketId, midPrice, bestBid, bestAsk, spread, orderbookDepth ({bids: number, asks: number}), baseDecimals, quoteDecimals
   - `ValidationResult` interface: valid (boolean), warnings (string[]), errors (string[]), adjustedProposal? (optional)
   - `StrategyGenerationRequest` interface: presetId (string|null), freeText (string), source ('preset'|'freetext'), marketId (string)
   - `StrategyGenerationResponse` interface: proposal (StrategyProposal from schemas), validation (ValidationResult), market ({midPrice, bestBid, bestAsk})

3. Create `src/lib/ai/schemas.ts`:
   - Import z from 'zod'
   - `strategyOrderSchema`: z.object with side (enum buy/sell), price (number positive), quantity (number positive), percentOfTotal (number 0-100). Add detailed `.describe()` on every field including units and examples.
   - `strategyProposalSchema`: z.object with strategyType (enum: 'dca', 'stop-loss', 'scale-in', 'take-profit', 'limit-buy', 'range-accumulate'), marketId (string), baseDenom (string), quoteDenom (string), orders (array of strategyOrderSchema, min 1 max 20), totalCapitalRequired (number positive), priceRange ({min: number positive, max: number positive}), splitCount (number int 1-20), worstCaseOutcome (string), reasoning (string). Add `.describe()` on ALL fields.
   - Export inferred types: `StrategyProposal = z.infer<typeof strategyProposalSchema>`, `StrategyOrder = z.infer<typeof strategyOrderSchema>`

4. Create `src/lib/ai/prompts.ts`:
   - Import MarketSnapshot from strategy/types
   - `PRESET_PROMPTS` record mapping all 6 preset IDs from `@/constants/intents.ts` ('dca-inj', 'stop-loss', 'scale-in', 'take-profit', 'limit-buy', 'range-accumulate') to specific strategy generation prompts. Each prompt must instruct the AI precisely about order structure (e.g., DCA = equal splits below market, scale-in = progressively larger at lower prices).
   - `buildSystemPrompt(market: MarketSnapshot): string` -- includes current market data (mid price, best bid, best ask, spread %), rules (prices must be realistic, buy below market, sell above, max 20 orders, respect budget, honest worst case), and instruction to NOT generate market IDs or denoms (they will be injected).
   - `getPromptForIntent(presetId: string | null, freeText: string, source: 'preset' | 'freetext'): string` -- returns preset prompt if source is preset, otherwise returns freeText (truncated to 500 chars).

5. Create `src/lib/ai/provider.ts`:
   - Import openai from '@ai-sdk/openai'
   - `getModel(source: 'preset' | 'freetext')` -- returns openai('gpt-4o-mini') for preset (simpler, cheaper), openai('gpt-4o') for freetext (needs more reasoning). Uses `OPENAI_API_KEY` env var (automatically picked up by @ai-sdk/openai).
   - Export the function, no hardcoded API keys.
  </action>

All files compile.

<task type="auto">
  <name>Task 2: Create Injective spot market service, strategy validator, and calculator</name>
  <files>
    src/services/injective/spot.ts

    src/lib/strategy/calculator.ts

  <action>
1. Create `src/services/injective/spot.ts`:
   - Import IndexerGrpcSpotApi from '@injectivelabs/sdk-ts' (check exact import path -- may be '@injectivelabs/sdk-ts/client/indexer' or just '@injectivelabs/sdk-ts')
   - Import getEndpoints from '@/services/injective/network'
   - Import MarketSnapshot from '@/lib/strategy/types'
   - Create indexer instance: `new IndexerGrpcSpotApi(getEndpoints().indexer)`
   - `fetchSpotMarkets()`: returns all spot markets from indexer
   - `fetchOrderbookSnapshot(marketId: string)`: returns orderbook via fetchOrderbookV2
   - `fetchMarketSnapshot(marketId: string): Promise<MarketSnapshot>`: fetches market + orderbook in parallel, computes midPrice from best bid/ask, spread as (bestAsk - bestBid)/bestAsk, returns full MarketSnapshot with baseDecimals and quoteDecimals from market metadata
   - `SUPPORTED_MARKETS` config: Record<string, {name: string, baseDenom: string, quoteDenom: string}>. For MVP, hardcode INJ/USDT. Market ID will be fetched dynamically.
   - `getSupportedMarkets()`: async function that fetches all markets, filters to SUPPORTED_MARKETS entries by matching base/quote denom symbols, returns map of readable name to market ID. Cache result in module-level variable.

2. Create `src/lib/strategy/validator.ts`:
   - Import StrategyProposal from schemas, MarketSnapshot and ValidationResult from types
   - `validateProposal(proposal: StrategyProposal, market: MarketSnapshot): ValidationResult`:
     - Check price range deviation from midPrice: >50% = error (AI-05: refuse), 20-50% = warning
     - Check buy orders are at/below bestAsk, sell orders at/above bestBid (warn if crossing spread)
     - Check totalCapitalRequired > 0
     - Check orders sum of (price * quantity) roughly matches totalCapitalRequired (warn if >10% discrepancy)
     - Check splitCount matches orders.length (warn if mismatch)
     - Return {valid: errors.length === 0, warnings, errors}
   - Risk threshold constants at top of file: MAX_PRICE_DEVIATION = 0.5, MAX_ORDERS = 20. Hardcoded for MVP per research recommendation.

3. Create `src/lib/strategy/calculator.ts`:
   - Import StrategyOrder and StrategyProposal from schemas
   - `recalculateOrders(params: {strategyType: string, priceRange: {min: number, max: number}, splitCount: number, totalAmount: number, side: 'buy' | 'sell'}): StrategyOrder[]`:
     - For DCA / range-accumulate / limit-buy: distribute totalAmount equally across splitCount price points, evenly spaced between priceRange.min and priceRange.max
     - For scale-in: distribute with increasing size at lower prices (weight: 1,2,3...N normalized)
     - For take-profit: distribute evenly across ascending price targets
     - For stop-loss: single order at priceRange.min
     - Each order gets percentOfTotal = (order capital / total) * 100
     - Returns array of StrategyOrder objects
   - This is the function used for client-side recalculation when user adjusts parameters (NO AI call needed).
  </action>

All compile without errors.
  </done>
