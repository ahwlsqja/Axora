  <action>
    1. Create `src/services/injective/wallet.ts`:
       - Import `walletStrategy` from `./client`, `Wallet` from `@injectivelabs/ts-types`, `getInjectiveAddress` from `@injectivelabs/sdk-ts`
       - `isWalletInstalled(walletType: Wallet): boolean` -- check `window.keplr` for Keplr, `window.ethereum` for MetaMask. Guard with `typeof window !== 'undefined'`.
       - `getInstallUrl(walletType: Wallet): string` -- return Chrome Web Store URLs for Keplr and MetaMask extensions.
       - `connectWallet(walletType: Wallet): Promise<string>` -- call `walletStrategy.setWallet(walletType)`, then `walletStrategy.getAddresses()`. For MetaMask, convert address with `getInjectiveAddress(addresses[0])`. For Keplr, use `addresses[0]` directly. Return the Injective address. Throw descriptive error if no addresses returned.
       - `disconnectWallet(): void` -- call `walletStrategy.disconnect()`.
       - Session persistence (Claude's discretion): save `{ walletType }` to `localStorage` key `'injective-agent-wallet'` on connect. Export `getSavedWalletType(): Wallet | null` to read it. Clear on disconnect.

    2. Create `src/services/injective/bank.ts`:
       - Import `ChainGrpcBankApi` from `@injectivelabs/sdk-ts`
       - Import `withFallback` from `./network` and `getEndpoints` from `./network`
       - `fetchBalances(injectiveAddress: string)`: use `withFallback` to try `new ChainGrpcBankApi(endpoint).fetchBalances(injectiveAddress)` across fallback endpoints. Return the balances array.
       - `fetchBalance(injectiveAddress: string, denom: string)`: similar pattern for single denom.

    3. Create `src/stores/walletStore.ts`:
       - Zustand store with state:
         ```
         address: string | null
         walletType: Wallet | null
         status: ConnectionStatus  // 'disconnected' | 'connecting' | 'connected'
         ```
       - Actions:
         ```
         connect(walletType: Wallet): Promise<void> -- sets status to 'connecting', calls connectWallet, sets address + status to 'connected', handles errors (toast via sonner, reset status)
         disconnect(): void -- calls disconnectWallet, resets all state to initial
         tryReconnect(): Promise<void> -- reads saved wallet type from localStorage, if exists, attempts silent reconnect. If fails, clears saved state (no popup, no error toast on auto-reconnect failure).
         ```

    4. Create `src/hooks/useWallet.ts`:
       - Thin wrapper exporting `useWallet()` that returns `{ address, walletType, status, connect, disconnect }` from useWalletStore.
       - On mount (useEffect), call `tryReconnect()` once. Guard with a ref to prevent double-call in StrictMode.

    5. Create `src/hooks/useBalances.ts`:
       - Import `useQuery` from `@tanstack/react-query`
       - Import `fetchBalances` from services
       - Import `BALANCE_POLL_INTERVAL`, `DISPLAY_DENOMS` from constants
       - `useBalances(injectiveAddress: string | null)`:
         - queryKey: `['balances', injectiveAddress]`
         - queryFn: fetch balances, then map to `TokenBalance[]` filtering only denoms in `DISPLAY_DENOMS`, converting amounts using decimals
         - enabled: `!!injectiveAddress`
         - refetchInterval: `BALANCE_POLL_INTERVAL` (10s)
         - staleTime: 5000
       - Return `{ balances, isLoading, error }`
  </action>
  <verify>

  </verify>

    Wallet connect/disconnect logic works with WalletStrategy. Zustand store manages wallet state. Balance hook polls every 10s via React Query. Session persistence via localStorage.
  </done>
