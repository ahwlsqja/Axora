  <action>
    1. Create `src/services/injective/subaccount.ts`:
       - `getAgentSubaccountId(injectiveAddress: string): string`:
         - Convert to eth address via `getEthereumAddress(injectiveAddress).toLowerCase()`
         - Append 24-char nonce suffix: `'0'.repeat(23) + '1'` (nonce 1, NOT 0)
         - Return full subaccount ID string
       - `fetchSubaccountBalances(injectiveAddress: string)`:
         - Use `IndexerGrpcAccountApi` with `withFallback` to call `fetchPortfolio(injectiveAddress)`
         - Return portfolio including subaccount balances
         - Note: use `fetchPortfolio` NOT deprecated `fetchSubaccountBalancesList`

    2. Create `src/services/injective/authz.ts`:
       - Import `MsgGrant` from `@injectivelabs/sdk-ts`
       - Import `SPOT_MSG_TYPES`, `DEFAULT_AUTHZ_EXPIRY_SECONDS` from constants
       - `createSpotAuthzGrants(granter: string, grantee: string): MsgGrant[]`:
         - Create one `MsgGrant.fromJSON()` per spot message type (4 total):
           - `/injective.exchange.v1beta1.MsgCreateSpotLimitOrder`
           - `/injective.exchange.v1beta1.MsgCreateSpotMarketOrder`
           - `/injective.exchange.v1beta1.MsgCancelSpotOrder`
           - `/injective.exchange.v1beta1.MsgBatchUpdateOrders`
         - Each grant: `{ messageType, grantee, granter, expiryInSeconds: DEFAULT_AUTHZ_EXPIRY_SECONDS }`
         - IMPORTANT: Must explicitly set `expiryInSeconds: 604800` (7 days). Default is 5 YEARS if not set.
         - Return array of 4 MsgGrant objects

    3. Create `src/services/injective/onboarding.ts`:
       - Import `MsgDeposit` from `@injectivelabs/sdk-ts`
       - Import `BigNumberInBase` from `@injectivelabs/utils`
       - Import `msgBroadcaster` (or `getMsgBroadcaster()`) from `./client`
       - Import `getAgentSubaccountId` from `./subaccount`
       - Import `createSpotAuthzGrants` from `./authz`
       - `executeOnboarding(injectiveAddress: string, depositAmountInj: number): Promise<string>`:
         - Generate subaccountId via `getAgentSubaccountId(injectiveAddress)`
         - Create deposit message: `MsgDeposit.fromJSON({ amount: { denom: 'inj', amount: new BigNumberInBase(depositAmountInj).toWei().toFixed() }, subaccountId, injectiveAddress })`
         - Create grant messages: `createSpotAuthzGrants(injectiveAddress, injectiveAddress)` (self-grant for Phase 1 per research open question -- agent address not yet available. This will be updated in Phase 4 when the API service has its own address.)
         - Batch all into single tx: `msgBroadcaster.broadcast({ msgs: [depositMsg, ...grantMsgs], injectiveAddress })`
         - Return transaction hash
         - Note in code comment: "Granting to self as placeholder. Phase 4 will use dedicated agent address."

    4. Create `src/stores/onboardingStore.ts`:
       - Zustand store with state:
         ```
         currentStep: OnboardingStep  // 'connect' | 'explain' | 'deposit'
         depositAmount: number  // default recommended value: 0.1 INJ (small amount for testnet)
         isExecuting: boolean
         txHash: string | null
         error: string | null
         ```
       - Actions:
         ```
         setStep(step: OnboardingStep): void
         nextStep(): void -- connect -> explain -> deposit
         prevStep(): void
         setDepositAmount(amount: number): void
         setExecuting(executing: boolean): void
         setTxHash(hash: string): void
         setError(error: string | null): void
         reset(): void
         ```

    5. Create `src/hooks/useOnboarding.ts`:
       - Combines onboardingStore + wallet state
       - `useOnboarding()` returns:
         - All onboarding store state and actions
         - `executeOnboardingFlow()`: async function that calls `executeOnboarding(address, depositAmount)`, handles loading/success/error states, shows toast on success/failure
         - `isWalletConnected`: boolean from wallet store
         - `canProceed`: computed boolean per step (step 1: wallet connected, step 2: always true, step 3: depositAmount > 0)

    6. Create `src/hooks/useSubaccountBalance.ts`:
       - React Query hook similar to useBalances but for the agent subaccount
       - Queries portfolio and filters for the agent subaccount nonce 1
       - Same polling interval as main balance hook
  </action>
